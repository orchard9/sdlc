Proposed Experience Upgrade

Fast Onboarding + Action-Oriented Control Tower

This mockup compresses "time-to-first-win" and makes the dashboard operationally decisive. Onboarding drives users to their first successful lifecycle transition, then lands them in a dashboard centered on focus, approvals, blockers, and next executable actions.

Onboarding Flow (first 15 minutes)

Guided path from zero context to first completed transition.

1

Choose Start Mode

New project or existing repo. Set expected operator mode.

2

Auto-Detect State

Scan for `.sdlc/`, stack hints, and missing prerequisites.

sdlc state
3

Pick Operating Style

Guided (step mode) vs Autonomous (run to HITL gate).

4

Define First Milestone Outcome

Capture one user-visible outcome and acceptance criteria.

5

Generate Starter Plan

Create milestone + first features + dependency order.

sdlc feature create <slug> --title "..."
6

Run First Directive with Guard Rails

Left panel shows live `next --json`; right panel executes action.

sdlc next --for <slug> --json
7

Checkpoint + Explain Why It Worked

Show exact files touched and what phase moved.

8

Hand Off to Dashboard

Pin a deterministic "Do Next" queue and start event feed.

Guided First-Run Panel

Single place to understand, execute, and validate the first directive loop.

feature: auth-login action: create_spec is_heavy: false

Execution Rail

01. write `.sdlc/features/auth-login/spec.md`

02. run `sdlc artifact draft auth-login spec`

03. run `sdlc next --for auth-login --json`
