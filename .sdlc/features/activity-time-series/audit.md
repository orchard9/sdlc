# Audit: activity-time-series

## Security surface analysis

This feature is entirely frontend-only. No new backend routes, no new data persistence, no auth changes, no server-side code. The security surface is minimal.

## Finding 1 â€” XSS via tooltip content (ACCEPTED / NOT APPLICABLE)

The tooltip renders `formatMs()` output, bucket time range strings, and static label strings. All values are derived from numeric computations (`Math.round`, `toFixed`) applied to telemetry event timestamps. No user-controlled string is rendered as HTML â€” React renders all tooltip content as text nodes. **No XSS risk.**

## Finding 2 â€” `Date.parse()` on untrusted `ts` field

`buildTimeSeries` calls `Date.parse(e.ts)` on the `ts` field of telemetry events. `Date.parse` on an invalid string returns `NaN`, which is checked (`!isNaN(ms)`) before the event is added to the timestamped array. If all events produce `NaN`, the function returns `null` and the chart renders the safe fallback message. **No crash or injection risk.**

## Finding 3 â€” ResizeObserver API availability

`ResizeObserver` is used to measure chart container width. It is supported in all modern browsers (Chrome 64+, Firefox 69+, Safari 13.1+) but not in IE 11. Since this project targets modern browsers and the existing codebase uses modern APIs throughout, **no polyfill is needed.** If `ResizeObserver` is absent, the chart falls back to the initial `chartWidth = 300` constant â€” it renders at a fixed width rather than crashing.

## Finding 4 â€” `useRunTelemetry('')` when skipFetch is true

When `RunActivityFeed` receives an `events` prop, it calls `useRunTelemetry('', false)`. The hook checks `if (!runId) return` which prevents any HTTP request. No empty-string URL request is ever made. **No network security concern.**

## Finding 5 â€” Information exposure in tooltip

The tooltip displays wall-clock durations (ms) of LLM, tool, subagent, and idle time. This information is already available in the `RunActivityFeed` event list below the chart. The tooltip adds no new information exposure beyond what is already visible. **Accepted.**

## Finding 6 â€” SVG injection via label text

X-axis labels are generated by `formatMs()`, which returns strings like `"0ms"`, `"1.4s"`, `"2m 30s"`. These are rendered via SVG `<text>` elements, which do not interpret HTML or script content. **No SVG injection risk.**

## Verdict

No security findings requiring remediation. All findings are either not applicable (no XSS vectors), correctly handled (NaN guard), or informational (browser compatibility, info exposure). The feature is ready to advance to QA.
