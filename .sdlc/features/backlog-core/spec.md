# Spec: backlog-core

## Problem

Autonomous agent runs frequently surface out-of-scope concerns — race conditions in middleware,
unbounded memory growth, cross-cutting debt — that have no capture path. Tasks require a feature
slug. Comments are feature-local. Advisory is for macro health. Escalation stops the run.
Result: concerns observed during execution are lost when the context closes.

## Solution

Add a `BacklogItem` primitive to `sdlc-core` with a flat-file store at `.sdlc/backlog.yaml`.
Items are captured immediately, classified by kind, and can be promoted to features when the
time is right.

## Data Model

```rust
// crates/sdlc-core/src/backlog.rs

#[derive(Debug, Clone, Copy, PartialEq, Eq, Serialize, Deserialize)]
#[serde(rename_all = "snake_case")]
pub enum BacklogKind {
    Concern,  // bug risk, race, security, correctness
    Idea,     // future enhancement, refactor opportunity
    Debt,     // known technical debt, accretion
}

#[derive(Debug, Clone, Copy, PartialEq, Eq, Serialize, Deserialize)]
#[serde(rename_all = "snake_case")]
pub enum BacklogStatus {
    Open,
    Parked,
    Promoted,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct BacklogItem {
    pub id: String,                         // B1, B2, B3... sequential, never recycled
    pub title: String,                      // required; complete sentence + component ref
    pub kind: BacklogKind,                  // concern | idea | debt
    pub status: BacklogStatus,              // open | parked | promoted

    #[serde(skip_serializing_if = "Option::is_none")]
    pub description: Option<String>,        // multi-line context; optional

    #[serde(skip_serializing_if = "Option::is_none")]
    pub evidence: Option<String>,           // file:line or fn name; optional but encouraged

    #[serde(skip_serializing_if = "Option::is_none")]
    pub source_feature: Option<String>,     // which feature run surfaced this

    #[serde(skip_serializing_if = "Option::is_none")]
    pub park_reason: Option<String>,        // required when parking; set by park()

    #[serde(skip_serializing_if = "Option::is_none")]
    pub promoted_to: Option<String>,        // feature slug; set by promote()

    pub created_at: DateTime<Utc>,
    pub updated_at: DateTime<Utc>,
}

#[derive(Debug, Clone, Serialize, Deserialize, Default)]
pub struct BacklogStore {
    #[serde(default)]
    pub items: Vec<BacklogItem>,
}
```

## ID Generation

IDs are sequential B-prefixed integers: B1, B2, B3. Generated by finding the max existing
numeric suffix and incrementing by 1. Never recycled (deleted items become parked/promoted,
not removed from the file).

## File Layout

```
.sdlc/backlog.yaml   — flat list: { items: [...] }
```

Storage follows the same atomic-write pattern as `advisory.yaml` and `escalations.yaml`:
`io::atomic_write` for all saves.

## Public API

```rust
impl BacklogStore {
    /// Load `.sdlc/backlog.yaml`. Returns empty default if absent.
    pub fn load(root: &Path) -> Result<Self>;

    /// Atomically save `.sdlc/backlog.yaml`.
    pub fn save(&self, root: &Path) -> Result<()>;

    /// Add a new item. Returns the new item with assigned ID.
    pub fn add(
        root: &Path,
        title: String,
        kind: BacklogKind,
        description: Option<String>,
        evidence: Option<String>,
        source_feature: Option<String>,
    ) -> Result<BacklogItem>;

    /// List items. If source_feature is Some, filter to that feature only.
    /// If status_filter is Some, filter to that status.
    pub fn list(
        root: &Path,
        status_filter: Option<BacklogStatus>,
        source_feature: Option<&str>,
    ) -> Result<Vec<BacklogItem>>;

    /// Get a single item by ID. Errors with BacklogItemNotFound if absent.
    pub fn get(root: &Path, id: &str) -> Result<BacklogItem>;

    /// Park an item. park_reason is required (non-empty). Errors if already promoted.
    pub fn park(root: &Path, id: &str, park_reason: String) -> Result<BacklogItem>;

    /// Mark an item promoted, recording which feature it was promoted to.
    /// Errors if already promoted or parked.
    pub fn mark_promoted(root: &Path, id: &str, feature_slug: &str) -> Result<BacklogItem>;

    /// Next sequential ID: max existing suffix + 1, or B1 if empty.
    fn next_id(items: &[BacklogItem]) -> String;
}
```

Note: `promote()` creates a feature — that logic lives in the CLI layer, not in `sdlc-core`.
The core only provides `mark_promoted()` to record the linkage after the CLI has created
the feature.

## Constraints

- No `unwrap()` — all error paths use `?` and `SdlcError`
- All file writes via `io::atomic_write`
- `serde(rename_all = "snake_case")` on all enums
- Optional fields use `#[serde(skip_serializing_if = "Option::is_none")]`
- Unit tests cover: add, list (unfiltered), list with source_feature filter,
  list with status filter, park, park requires non-empty reason, mark_promoted,
  get, get missing ID, and round-trip serialization

## Out of Scope

- Feature creation (CLI responsibility)
- Milestone assignment (CLI responsibility)
- Source_feature auto-inference from state.yaml (CLI responsibility)
- Deduplication / fuzzy matching (v2)
- Priority / tags / source_run (v2)
