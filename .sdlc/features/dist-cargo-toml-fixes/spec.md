# Spec: cargo-dist: switch to musl Linux target and set install-path to ~/.local/bin

## Problem

The root `Cargo.toml` contains a `[profile.dist]` profile used for release builds. When cargo-dist or the release workflow produces Linux binaries, the target should be `x86_64-unknown-linux-musl` (and `aarch64-unknown-linux-musl`) to produce fully static binaries with no glibc dependency.

Additionally, when users run the install script generated by cargo-dist or the release workflow, the binary should be placed in `~/.local/bin` rather than the default location, so it lands on the PATH for most Linux users without requiring root.

## Current State

- `Cargo.toml` has a `[profile.dist]` section but no `[workspace.metadata.dist]` section configuring cargo-dist targets or install-path.
- The `.github/workflows/release.yml` already targets `x86_64-unknown-linux-musl` and `aarch64-unknown-linux-musl` directly via the matrix, but this is a custom build workflow, not a cargo-dist managed workflow.
- There is no `install-path` configuration in any Cargo.toml.

## Goal

Add `[workspace.metadata.dist]` (or equivalent cargo-dist config) to `Cargo.toml` that:

1. Specifies Linux targets as musl variants:
   - `x86_64-unknown-linux-musl`
   - `aarch64-unknown-linux-musl`
2. Sets `install-path = ["~/.local/bin"]` so the installer places the binary in `~/.local/bin`.

If the project is not using cargo-dist and instead uses a fully custom release workflow, update the release workflow and any installer-related documentation to reflect musl targets and the `~/.local/bin` install location.

## Acceptance Criteria

- `Cargo.toml` contains a `[workspace.metadata.dist]` section (or equivalent) with:
  - `targets` including `x86_64-unknown-linux-musl` and `aarch64-unknown-linux-musl`
  - `install-path` set to `["~/.local/bin"]`
- The release workflow continues to produce working static Linux binaries.
- No regressions to existing build, test, or lint CI steps.

## Out of Scope

- Changing the macOS or Windows release targets.
- Setting up cargo-dist from scratch if not already used — only configuration changes.
- Modifying `[profile.dist]` LTO or strip settings.
