crates/sdlc-cli/src/cmd/init/commands/sdlc_next.rs SDLC_NEXT_COMMAND 
For **artifact creation** (`create_spec`, `create_design`, ...):
1. Run `sdlc feature show <slug> --json` for context
2. Read existing artifacts in `.sdlc/features/<slug>/`
3. Write a thorough Markdown artifact to `output_path`

For `create_design` on a **UI feature**, also write `mockup.html`:
- Single self-contained file — inline `<style>` and `<script>`, no external resources
- Valid HTML5 with a `<nav>` or tab bar to navigate between screens
- Named sections (`<section id="screen-*">`) for each major UI state
- Representative colors and typography; pixel-perfect fidelity not required
- Reference it from `design.md`: `[Mockup](mockup.html)`
- Place the file in the same directory as `design.md`

Non-UI features (backend, CLI, config): mockup is optional.

HTML Mockup Format Contract

Rules every agent must follow when producing mockup.html.

ConstraintRationale
mockup.html — single file Committable to git, viewable by double-click, no server needed
Inline <style> block No external CSS dependency
Inline <script> block (if used) Screen navigation without a bundler
<!DOCTYPE html> + <html lang="en"> Valid HTML5; renders in all browsers
<nav> or tab bar at the top Reviewers navigate without scrolling through the whole file
<section id="screen-*"> per view Predictable structure; easy to reference in design.md
No CDN references (https://cdn.*) Offline-friendly; works in air-gapped environments
Placed in .sdlc/features/<slug>/ Alongside design.md; relative link works from the same directory
Minimal Structure
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8" />
  <title>Mockup: <feature-title></title>
  <style>
    /* all CSS inline here */
  </style>
</head>
<body>
  <nav>
    <button onclick="show('screen-a')">Screen A</button>
    <button onclick="show('screen-b')">Screen B</button>
  </nav>

  <section id="screen-a">
    <!-- first screen content -->
  </section>

  <section id="screen-b" style="display:none">
    <!-- second screen content -->
  </section>

  <script>
    function show(id) {
      document.querySelectorAll('section').forEach(s => s.style.display = 'none');
      document.getElementById(id).style.display = 'block';
    }
  </script>
</body>
</html>
