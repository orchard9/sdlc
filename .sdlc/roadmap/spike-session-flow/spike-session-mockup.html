Can we use sqlx for type-safe queries without a migration runner?

Awaiting verdict

Investigation complete. My recommendation is ADOPT.

sqlx's offline mode compiles queries against a cached schema snapshot — no live DB needed at build time. The compile-time check is real and catches column name/type mismatches. You don't need sqlx-migrate; the offline cache file (sqlx-data.json) is standalone.

The prototype at /tmp/spike-sqlx/prototype/ has a working example against our schema. I tested concurrent queries and error paths — both clean.

What's the maintenance burden of keeping sqlx-data.json in sync when the schema changes?

J

You run cargo sqlx prepare after schema migrations — it regenerates the cache. CI enforces freshness with cargo sqlx prepare --check. It's one extra step per migration, not per query. The burden is low and the compile-time guarantee is worth it.
