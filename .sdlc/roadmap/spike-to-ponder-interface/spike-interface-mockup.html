ADAPT 2026-03-01

playwright-mcp-uat

We need browser automation for milestone acceptance tests so that UAT runs are deterministic and repeatable without human interaction.

findings.md .sdlc/spikes/playwright-mcp-uat/findings.md
The Question

Can Playwright MCP provide reliable browser automation for milestone UAT runs within Claude agent sessions without requiring a persistent browser service?

Success Criteria

Agent can navigate to localhost UI, interact with elements, and capture screenshot evidence in a single agent session without external service coordination.

Candidates Evaluated
Candidate Verdict Reason
Playwright MCP adapt Works but needs screenshot/DOM capture protocol extension for evidence collection
Selenium Grid eliminated Requires persistent external service — defeats self-contained deployment
Risks and Open Questions
  • • MCP tool list injection per-run: how do we ensure UAT runs get Playwright tools without leaking to other agent types?
  • • Screenshot artifact persistence: agent captures screenshots to /tmp — we need a mechanism to copy evidence to .sdlc/milestones/<slug>/uat/
  • • Browser install on CI: npx playwright install needs to run in the pipeline; no current mechanism for this in woodpecker config
  • • Port binding conflicts: UAT agent needs to know what port sdlc-server is running on at test time
